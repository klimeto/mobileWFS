Home window

After the first launch, the home window is divided in three main blocks: User Login placed at the top, Observations Management placed below, and the Main menu placed in the top right corner of the App home window. A fourth block will appear once the first observation/s is added/updated or removed, this is the Push data to server.

User Management

CHANGE USER

The first launch of the app comes with predefined user DEMO with its PIN 0. The app can be tested using the DEMO#0 user. If you wanna create a new or SIGN IN with previously created user just click on the button.

Nickname definition

The app will create a new nickname, if IT does not exist yet, and a unique number (PIN) is assigned to you (ex. 84) after which you are instantly signed in. Each time you will launch the app you will be automatically logged in with your newly created nickname (proplantTester#84). If you use the App in a different device you can use the same nickname and log in.

Observations Management

New observations

Under the Observation management Module we have the New observation/s, where a list of observation/s collected on the field is created each time. In the new observation list all observations collected that did not exist before and could be potentially pushed to the proplant database server, to be seen by each user, once you finalized your field observation. Once it is pushed to the server the new observation list will be rest to zero. Each observation block provides the data you collected as e.g. Plant name, Observation date, Certainity of your data, Latitude and Longitude and a toolbox. The toolbox offers tools as open the editor window, zoom on observation within the map window and delete that particular observation. The same tools are provide for each type of observation you're the owner (creator).

Updated observations

Updated observations are all those new observations you have created and pushed to the proplant data server before, and at the present time you are updating them.

Removed observations

Removed observations are agin those already uploaded to the proplant data server and currently being deleted.

Main menu

Local data

Local data item will load all the data available in the app local database and display them on the map. The observations (map markers) have 2 diffrerent type of to distinguish whether the observation is from you or from different user. The yellow marker represents observations collected by you; and the blue marker collected by another user.

Server data

Server data item will load all the data available in the proplant data server into the app local database and display them on the map. The observation markers will have different colours as explained in previous point.

Open map

Open map item will open the map window, with or without loaded the observations.

Push data to server

Overall statistics

The overall statistics block provides a summary of observations you have collected as new / updated / removed as well as the number of photographs taken.

Terms and conditions

Tapping the text You agree with our Terms and conditions will open the Terms of use window describing dataset license.

Discard all

This button helps to eliminate all the edits that the user made on the observations.

Push all

This button allows to send to the server (in a push) all the new observation/s or edits/updates the user makes, uploading at the same time the photos taken of the plants. The observations will be sent to the proplant database server and will be available for all users to display and use using OGC WMS and WFS.
